 - 클린 코드의 중요성(나쁜 코드는 나쁜 코드를 더 많이 양산한다)

  클린 코드의란?
  1. 보기에 즐거운 코드
  2. 단순하고 직접적이여서 가독성이 좋은 코드
  3. 다른사람이 고치기 쉬운 코드
  4. 주의 깊게 작성한 코드

  간단한 코드란?
  1. 모든 테스트를 통과한다.
  2. 중복이 없다.
  3. 시스템 내 모든 설계 아이디어를 표현한다.
  4. 클래스, 메서드, 함수 등을 최대한 줄인다.
